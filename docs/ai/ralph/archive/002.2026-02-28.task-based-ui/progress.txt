## Codebase Patterns

- The bookmarklet (`js/script.js`) is a single-file IIFE starting with `javascript:`. All HTML is string concatenation (no template literals). CSS is inline styles.
- Playwright tests for the bookmarklet use a test harness (`js/tests/harness.html`) and inject the bookmarklet by stripping the `javascript:` prefix and evaluating the code in the page context.
- The bookmarklet UI object is `UIImpl` (assigned to `UI` global). Key methods: `inject()`, `setStatus()`, `setBar()`, `renderAll()`, `renderLogs()`, `ensureTick()`.
- `renderAll()` is called on a 1-second interval timer (`ensureTick`) and also manually after state changes. It updates stat counters and the progress bar.
- The panel uses `data-testid` attributes for test selectors on container elements (e.g., `data-testid="cvz-stats"`, `data-testid="cvz-controls"`).
- `#cvz-bar` is an inline thin progress bar inside the stats container (not standalone). It has `width:0%` by default, so Playwright considers it invisible - test the parent track element visibility instead.
- The `TaskList` object is exposed on `window.__cvz_TaskList` for testing. `UIImpl` is exposed on `window.__cvz_UI`. `Exporter`, state (`S`), and `Net` are exposed as `window.__cvz_Exporter`, `window.__cvz_S`, `window.__cvz_Net`. All are accessible from Playwright `page.evaluate()`.
- `TaskList.render()` uses a dirty flag pattern - only re-renders when `_dirty` is true. `add()` and `update()` set `_dirty = true`. `render()` clears it.
- Task detail sub-lines (e.g., "downloading 2/5 files") are rendered as sibling `<div>` elements with class `cvz-task-detail`, not nested inside the task's main `<div>`. Use `.locator(".cvz-task-detail")` to find them, not by looking inside `.cvz-task-active`.
- For tests that call `Exporter` methods (rescan, exportOneBatch), the page must be served from `chatgpt.com` origin to pass `assertOnChatGPT()`. Use `page.route("https://chatgpt.com/...", ...)` with Playwright route interception. Also mock all `/backend-api/` routes to avoid real network calls.

- Active tasks keep their last status when `Exporter.stop()` is called (not forcibly updated)

- Files changed:
  - `js/script.js` - Wired TaskList to exportOneBatch, exportKnowledgeBatch, rescan; exposed internals for testing
  - `js/tests/export-pipeline.spec.js` - 14 Playwright tests for US-003

- **Learnings for future iterations:**

  - Tests that exercise `Exporter.rescan()` or `Exporter.start()` must run from a `chatgpt.com` origin because `assertOnChatGPT()` checks `location.hostname`. Use Playwright `page.route()` to serve the harness from `https://chatgpt.com/test-harness`.
  - The bookmarklet triggers auto-rescan 800ms after injection when running on chatgpt.com. Tests need to mock all API routes before navigating and wait ~1500ms for auto-rescan to complete before running test-specific operations.
  - The initial "fetching conversation" detail is set in `TaskList.add()` (not `update()`). To capture all detail changes in tests, monkey-patch both `add()` and `update()`.
