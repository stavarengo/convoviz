# PRD: Persistent Structured Logging for JS Exporter

## Codebase Patterns

Squash from: 2f13fe966bb53ddfe407527041892123b2b0e8d1

- IDB stores use factory functions (e.g., `createLogStore()`, `createDiscoveryStore()`) that return an object with `init()` and operation methods. The `init()` opens the database; all operations are no-ops if `init()` hasn't been called or failed.
- `fake-indexeddb/auto` shares IDB state across tests in the same file. Each `beforeEach` must clear the database via `init()` + `clear()` + `vi.resetModules()` to isolate tests.
- Tests use dynamic imports (`await import(...)`) combined with `vi.resetModules()` to get fresh module state per test.
- Quality gate: `npm run typecheck` (tsc --noEmit) + `npm test` (vitest run) must both pass.
- The logger module (`logger.ts`) uses module-level state for session ID and in-memory log array. Use `vi.resetModules()` + dynamic imports in tests to get fresh state per test. `initLogger()` must be called to enable IDB persistence; without it, `log()` still works in-memory only.
- All dependency interfaces use `log: (level: LogLevel, category: string, message: string, context?: Record<string, unknown>) => void` â€” import `LogLevel` from `state/logger`. Mock with `vi.fn()` in tests.
- `ExportState` no longer has a `logs` field. Logs are decoupled from export state entirely. The `renderLogs()` method in panel.ts is a stub pending US-005 wiring.
- Categories: `sys`, `scan`, `chat`, `file`, `kf`, `net`, `ui`, `state` â€” follow these consistently.
- Zero `console.error`/`console.warn` calls in source. Low-level modules (`store.ts`, `export-blobs.ts`) import `log` directly from `logger.ts`. `bus.ts` takes an optional `onError` callback to stay decoupled.
- `vi.doMock()` must be called AFTER `vi.resetModules()` â€” if called before, the mock registration is cleared by resetModules.
