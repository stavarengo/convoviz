## Codebase Patterns

- Playwright tests for the bookmarklet use `js/tests/export-pipeline.spec.js` with a shared `beforeEach` that sets up route mocks for chatgpt.com APIs
- The bookmarklet state (`S`) is exposed as `window.__cvz_S`, TaskList as `window.__cvz_TaskList`, Exporter as `window.__cvz_Exporter`, Net as `window.__cvz_Net`
- `addLog` is not directly exposed; read `S.logs` to inspect log output in tests
- Route overrides in Playwright can be stacked; later `page.route()` calls for the same pattern override earlier ones within a test
- `S.projects` structure: `[{ gizmoId, name, files: [{ fileId, name, type, size }] }]`
- The bookmarklet uses `function(){}` syntax (not arrows) in the `exportOneBatch` area -- match this style
- `js/script.min.js` is auto-generated by pre-commit hook (terser) -- never edit manually
- 2 scan tests and 1 version string test are pre-existing failures in the test suite (not related to this PRD)
- No `package.json` is committed; `npm init -y && npm install --save-dev @playwright/test && npx playwright install chromium` is needed in fresh worktrees
