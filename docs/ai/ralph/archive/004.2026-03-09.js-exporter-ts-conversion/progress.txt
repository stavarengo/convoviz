## Codebase Patterns

Squash from: b72c7af67165253769258b312614e03f6c6eecb1

- Root `.gitignore` has `/package.json` and `/package-lock.json` (root-only); `node_modules/` is global. `js-exporter/package.json` is NOT ignored.
- `js-exporter/` uses npm (not yarn/pnpm/bun) per PRD requirement.
- `vitest.config.ts` needs `passWithNoTests: true` to exit 0 when no test files exist.
- `tsconfig.json` needs at least one `.ts` file in `src/` or `tsc --noEmit` fails with TS18003. `src/index.ts` with `export {};` serves as placeholder.
- Empty directories tracked via `.gitkeep` files (remove when real files are added).
- When testing async rejection with fake timers, attach `.catch((e) => e)` before `vi.advanceTimersByTimeAsync()` to avoid unhandled rejection warnings.
- All shared types live in `src/types.ts`. Import with `import type { ... } from "../types"` from module files, or `"./types"` from `src/` root.
- `kfExported` items share the `KfPendingItem` shape (no extra fields). `DeadItem` and `KfDeadItem` extend their pending counterparts with `lastError`.
- `tsconfig.json` only includes `src/`; test files are not typechecked by `npm run typecheck`. Vitest handles test compilation at runtime.
- `Uint8Array` spread into `Blob` constructor fails TS strict typecheck. Cast with `as BlobPart[]`.
- When dynamically assigning properties via `for..in` on a typed object, use double cast: `(obj as unknown as Record<string, unknown>)[k]`.
- jsdom normalizes CSS values (e.g., `0.0%` becomes `0%`). Use `parseFloat(el.style.width)` in tests instead of string-matching.
- jsdom's `Blob` may lack `arrayBuffer()`. When testing code that calls `blob.arrayBuffer()`, polyfill it via `FileReader` or use a `makeBlob()` helper in tests.
- `main.ts` IIFE auto-executes on import. In tests, stub `alert` and `console.error` before importing to suppress jsdom noise. The IIFE fails gracefully (assertOnChatGPT throws on localhost).
- `Store.load()` already calls `mergeState()` internally; do not double-call `mergeState()` on the result.
