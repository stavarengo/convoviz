# PRD: Event-Driven Exporter Architecture

Base commit: 49d319072cdd86fc5e73168ba7568fbe05c2e9aa

## Codebase Patterns

- JS exporter uses `vitest` for tests, `fake-indexeddb` for IDB mocking, `jsdom` for DOM
- Tests live in `js-exporter/tests/` mirroring the `src/` directory structure
- Factory pattern is standard: `createQueue()`, `createEventBus()`, `createDiscoveryStore()`, etc.
- Types are centralized in `src/types.ts`; module-specific types are co-located
- `npx tsc --noEmit` for typecheck, `npx vitest run` for tests
- Must `npm install` in `js-exporter/` before running tests in worktrees
- EventBus listeners are called synchronously; async listeners' promises are not awaited. Test async listeners by calling the function directly or collecting promises.
- Bootstrap wiring tests use `vi.waitFor()` to handle async event listeners (discovery store lookups are async even though EventBus emit is synchronous)
- Chat worker emits `conversation-files-discovered` and `conversation-exported` events instead of calling `attachmentQueue.enqueue()` directly; the bootstrap wires these events to queues
- The coordinator (`src/export/coordinator.ts`) is a thin lifecycle orchestrator; all scan logic is in scanners, all queue creation in `bootstrap.ts`
- Queue completion is detected via the Promise returned by `queue.start()` resolving when the queue drains
- Bootstrap exposes `setScanAbortSignal()` and `getSpawnedScannerPromises()` for coordinator control of project-spawned scanners
- `main.ts` initializes the discovery store, calls `bootstrap()`, then `createCoordinator()` -- the entry point wiring

- **Learnings for future iterations:**
