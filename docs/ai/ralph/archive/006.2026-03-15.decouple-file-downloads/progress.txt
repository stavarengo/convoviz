# PRD: Decouple File Downloads from Chat Export

Base commit: d743f3c67790bd08acc71e2320ca2ca39c5354ba

## Codebase Patterns

Squash from: d743f3c67790bd08acc71e2320ca2ca39c5354ba
Final commit: 475e00a9ab4db1a5f571fb30e719774253ae71ec

- js-exporter uses vitest for tests, tsconfig with `noEmit: true`, esbuild for bundling
- Test files live in `js-exporter/tests/` mirroring `js-exporter/src/` structure
- Use `npx vitest run` and `npx tsc --noEmit` for quality checks (no global installs)
- Need `npm install` in worktrees before running tests (node_modules not shared)
- Factory function pattern: `createQueue()` not `new Queue()` â€” consistent with `createExporter()`, `createChatWorker()`
- Queue auto-drain requires `itemsEverSeen` flag â€” an empty start() must park workers, not resolve immediately
- Worker functions follow closure-based DI: `createXWorker(deps)` returns `(item, signal) => Promise<void>`
- `AttachmentItem` type defined in `attachment-worker.ts` but imported by `chat-worker.ts` â€” shared across workers
- `ExportBlobStore.hasFilePrefix(prefix)` checks if any file key in IDB starts with the given prefix (used for attachment dedup)
- Queue supports `immediateDeadLetter` property on thrown errors â€” sets fail count to maxRetries, bypassing retry requeue
- Exporter uses a coordinator pattern for completion detection: chatDrained + attachmentIdle + knowledgeDrained = all done
- Attachment queue starts with all queues but receives items dynamically from chat workers; stop it when chat drains and attachment is idle
- `Exporter` interface no longer has `exportOneBatch`/`exportKnowledgeBatch` â€” replaced by queue-based processing
- Exporter exposes `chatQueue`, `attachmentQueue`, `knowledgeQueue` on the instance for UI to call `setConcurrency()`
- State v3: `Settings` has `chatConcurrency`, `fileConcurrency`, `knowledgeFileConcurrency` (no more `batch`/`conc`)
- State v3: `Progress` KF fields renamed `kf*` -> `knowledgeFiles*`, new `file*` fields for attachment queue
- State v3: `Stats` uses `chatsExported`/`chatsMs`/`filesDownloaded`/`filesMs`/`knowledgeFilesDownloaded`/`knowledgeFilesMs`
- State v3: `RunState` no longer has `lastPhase`
- Migration function lives in `state/migrate.ts` (separate from `store.ts` and `defaults.ts` to avoid circular imports)
- `scan/conversations.ts` uses fixed page size of 100 (no longer derived from batch setting)

- **Learnings for future iterations:**
  - The attachment queue cannot use the standard queue drain detection when it starts empty (no items = `itemsEverSeen` stays false, queue parks forever). Use a coordinator pattern: start the attachment queue, and stop it externally when the chat queue drains and the attachment queue is idle.
